# Impeccable — moved

This project now uses the standard Impeccable context files at the repo root:

- **[PRODUCT.md](PRODUCT.md)** — strategic context: register, users, product purpose, brand personality, anti-references, design principles, accessibility.
- **[DESIGN.md](DESIGN.md)** — visual system: theme, color tokens, typography, spacing/shape, components, layout, per-portal tuning, RTL-readiness.

Read those two files before any design or frontend work on the **operational portals** (Admin / Owner / Client). The public storefront keeps its own warm-terracotta theme and is out of scope.
